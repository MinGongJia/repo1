一、静态网站和动态网站
	静态网站：所有的页面是固定不变的，无法及时地更新数据。
	动态网站：页面和服务器进行交互，获得服务器端动态生成的数据，在页面中呈现。

二、ASP.NET
	1..NET：开发平台，核心框架：.NET Framework
	ASP.NET是基于.NET Framework框架，开发动态网站的技术
	2.ASP.NET(Active Server Page)是一种动态的网页技术，基于.NET Framework平台，在服务器端运行后台代码，动
	态生成HTML。
	常见的动态网页技术：ASP.NET,JSP,PHP
	3.ASP.NET技术包括两个方向：
		1.web form
		控件 + 事件驱动
		2.MVC
三、C/S与B/S的区别
	1.C/S(Client/Server)：客户端服务器架构，客户端安装软件，业务处理在客户端及其上完成
	2.B/S(Browser/Server)：浏览器服务器架构，客户端只需要有浏览器，业务处理在服务器端完成，对服务器要求
	比较高

四、服务器端
	1.硬件；服务器电脑
	2.软件：IIS
		IIS(Internet Infomation Service)因特网信息服务：在Windows操作系统下对网站进行管理的软件。
		在VS中，内置了轻量级的Web服务器
			IIS Express
			无需配置IIS、设置相应权限

五、新建一个ASP.NET web应用程序
	1.添加web窗体
		扩展名为.asp的文件：前端页面，
			可以编写html代码，css样式，js/jquery脚本
			还可以编写asp.net服务端控件(ID,runat="server")：运行在服务端的控件
		扩展名为.aspx.cs的文件：后台代码，可以编写C#代码

	ASP.NET有两种编码形式：代码内嵌和代码后置
		代码内嵌：HTML代码和C#代码交叉使用

六、B/S中浏览器端与服务器端采用请求/响应模式进行交互
	在浏览器中输入页面地址项服务器请求页面，服务器端接收到请求后会进行服务端应用程序的运行，有可能还要去
访问数据库服务器，处理完成后，服务器将处理好的结果返回给浏览器端，此时，不管当初请求时扩展名为aspx的页面
，此时都会渲染成普通的html标签，有浏览器去解析呈现，用户看到的页面就是普通的html页面。

七、aspx页面的请求过程
	1.首先是来自于客户端浏览器的请求，如：http://localhost:44322/WebForm1.aspx ,该请求发送给IIS服务器
	2.web服务器IIS接收到请求后，检查请求的页面类型，如果是普通的html页面，IIS直接将该页面从服务端发回到
	客户端，由浏览器去解析；如果请求的是扩展名为aspx页面，IIS处理不了这个请求，它会将这个请求交给专门的
	asp.net引擎(aspnet_isapi.dll)处理
	3.asp.net引擎会将后台代码交给CLR运行，执行完之后再将aspx和aspx.cs两个文件合并成一个页面类，并将页面类
	实例化成一个页面对象发回到客户端，此时，发回的页面中只有一些html标签，css样式，Js脚本，由浏览器解析呈
	现。
		第一次获取页面较慢，第二次获取较快。也为页面类代码不需要重新生成，直接实例化。

系统对象：
	在ASP.NET页面中包含一些类，在页面中可以直接使用，我们称之为系统对象。
	· Page对象：是指页面自身的方式，在整个页面执行期内，都可以使用该对象。
	· Request对象：此对象封装了由Web浏览器或其他客户端生成的HTTP请求的细节。主要提供从浏览器读取信息或
	读取客户端信息等功能，用于页面请求期。
	· Response对象：此对象封装了返回到HTTP客户端的输出，提供向浏览器输出信息或者发送指令，用于页面执行期。
	· Server对象：该对象提供了服务端的一些属性方法，比如页面的绝对路径等。
	· Application对象：为所有用户提供共享信息，作用于真个应用程序运行期。
	· Session对象：为某个用户提供信息，作用于用户会话期。
	· Cookie对象：保持与客户端的共享信息方式。
